**Create `apps/mesh/src/diagnostic/orchestrator.ts`** — The core orchestration engine:

```typescript
interface RunDiagnosticOptions {
  url: string;
  storage: DiagnosticSessionStorage;
  force?: boolean;  // Skip cache, force re-scan
}

async function runDiagnostic(options: RunDiagnosticOptions): Promise<{ token: string; cached: boolean }>
```

Implementation:

**Step 1: URL validation**
- Call `validateUrl(url)` from the SSRF validator
- If validation fails, throw a descriptive error (the route handler catches and returns 400)

**Step 2: Cache check**
- If `force` is not true, call `storage.findRecentByNormalizedUrl(normalizedUrl, 24 * 60 * 60 * 1000)` (24 hours)
- If a completed session exists → return `{ token: existingSession.token, cached: true }`
- This satisfies the "cache with re-scan option" decision

**Step 3: Create session**
- Create a new session via `storage.create(...)` with:
  - Status: `"pending"`
  - Agents: all 4 initialized to `{ status: "pending" }`
  - Results: `{}`
  - expiresAt: 7 days from now
- Return the token immediately to the caller

**Step 4: Run agents in background (fire-and-forget)**
- After returning the token, kick off `executeAgents(token, url, storage)` as a detached promise (do NOT await it)
- Use `Promise.resolve().then(() => executeAgents(...)).catch(...)` to prevent unhandled rejection

**`executeAgents(token, url, storage)`** — Internal function, NOT exported:

1. Update session status to `"running"`
2. Crawl homepage: `const crawl = await crawlPage(url)` with 30s timeout
   - If crawl fails → update all agents to `"failed"`, set session status to `"failed"`, return early
3. Run all 4 agents in parallel using `Promise.allSettled`:

```typescript
const agentTasks = [
  {
    id: "web_performance" as const,
    run: () => runWebPerformanceAgent(url, crawl),
    timeoutMs: 90_000,  // 90s — PSI can be very slow
  },
  {
    id: "seo" as const,
    run: () => runSeoAgent(crawl),
    timeoutMs: 30_000,  // 30s
  },
  {
    id: "tech_stack" as const,
    run: () => runTechStackAgent(crawl),
    timeoutMs: 30_000,  // 30s
  },
  {
    id: "company_context" as const,
    run: () => runCompanyContextAgent(url, crawl),
    timeoutMs: 60_000,  // 60s — multi-page crawl + LLM
  },
];
```

4. For each agent, wrap in a timeout + status tracking:

```typescript
async function runAgentWithTracking(agent, token, storage) {
  await storage.updateAgentStatus(token, agent.id, {
    status: "running",
    startedAt: new Date().toISOString(),
  });

  try {
    const result = await Promise.race([
      agent.run(),
      new Promise((_, reject) =>
        setTimeout(() => reject(new Error("Agent timeout")), agent.timeoutMs)
      ),
    ]);

    await storage.updateAgentStatus(token, agent.id, {
      status: "completed",
      startedAt: /* preserve */,
      completedAt: new Date().toISOString(),
    });

    await storage.updateResults(token, agent.id, result);
  } catch (error) {
    await storage.updateAgentStatus(token, agent.id, {
      status: "failed",
      startedAt: /* preserve */,
      completedAt: new Date().toISOString(),
      error: error instanceof Error ? error.message : "Unknown error",
    });
    // Do NOT rethrow — partial results are expected
  }
}
```

5. Run all agents via `Promise.allSettled(agentTasks.map(a => runAgentWithTracking(a, token, storage)))`
6. After all agents finish (settled), update session status to `"completed"` (even if some failed — per user decision "always produce a report")
7. Log completion: `console.log("[diagnostic] Scan complete for ${url}: ${completedCount}/${totalCount} agents succeeded")`

**Rate limiting** (Claude's discretion per CONTEXT.md):

```typescript
const rateLimiter = new Map<string, number>();  // IP → last request timestamp
const RATE_LIMIT_MS = 10_000;  // 10 seconds between scans per IP

function checkRateLimit(ip: string): boolean {
  const last = rateLimiter.get(ip) ?? 0;
  if (Date.now() - last < RATE_LIMIT_MS) return false;
  rateLimiter.set(ip, Date.now());
  // Cleanup old entries periodically
  if (rateLimiter.size > 10_000) {
    const cutoff = Date.now() - RATE_LIMIT_MS * 2;
    for (const [key, time] of rateLimiter) {
      if (time < cutoff) rateLimiter.delete(key);
    }
  }
  return true;
}
```

**POST `/scan`** — Trigger a new diagnostic scan:
- Request body: `{ url: string; force?: boolean }`
- Validate body with Zod: `z.object({ url: z.string().min(1).max(2000), force: z.boolean().optional() })`
- Extract client IP from `c.req.header("x-forwarded-for")` or `c.req.header("x-real-ip")` or `"unknown"`
- Call `checkRateLimit(ip)` — if rate-limited, return `429 { error: "Too many requests. Please wait before scanning again." }`
- Get database from the app context (will be injected when mounting — see app.ts changes below)
- Create `DiagnosticSessionStorage` instance from the database
- Call `runDiagnostic({ url, storage, force })` — this returns immediately with token
- On success, return `200 { token, cached }`
- On validation error (SSRF, bad URL), return `400 { error: message }`
- On unexpected error, return `500 { error: "Internal server error" }`

**GET `/session/:token`** — Poll session status:
- Extract `token` from path params
- Get database, create storage instance
- Call `storage.findByToken(token)`
- If not found → `404 { error: "Session not found" }`
- Return `200` with the session data:
```json
{
  "token": "...",
  "url": "...",
  "status": "running",
  "agents": {
    "web_performance": { "status": "running", "startedAt": "..." },
    "seo": { "status": "completed", "startedAt": "...", "completedAt": "..." },
    "tech_stack": { "status": "completed", "startedAt": "...", "completedAt": "..." },
    "company_context": { "status": "pending" }
  },
  "results": {
    "seo": { ... },
    "techStack": { ... }
  },
  "createdAt": "...",
  "updatedAt": "..."
}
```

The client polls this endpoint. Per CONTEXT.md: "Results reveal all at once after the full pipeline finishes" — the UI handles that. The API always returns current state including partial results.
