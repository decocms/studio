**Create `apps/mesh/src/diagnostic/crawl.ts`** — Shared HTML crawler used by all agents:

```typescript
interface CrawlResult {
  url: string;
  html: string;
  headers: Record<string, string>;
  statusCode: number;
  redirectedUrl?: string;  // Final URL after redirects
}

async function crawlPage(url: string, options?: { timeoutMs?: number }): Promise<CrawlResult>
```

Implementation:
- Use `fetch()` with a 30-second timeout (generous per user decision)
- Set a realistic User-Agent header: `"Mozilla/5.0 (compatible; MeshDiagnostic/1.0; +https://decocms.com)"` — avoid bot-blocking
- Follow redirects (default fetch behavior)
- Extract response headers into a plain Record
- Return HTML body as string
- If fetch fails or times out, throw a descriptive error
- Do NOT catch errors — let the orchestrator handle failures

Also export a helper:
```typescript
async function crawlMultiplePages(urls: string[], options?: { timeoutMs?: number; maxPages?: number }): Promise<CrawlResult[]>
```
- Crawl up to `maxPages` (default 5) URLs in parallel using `Promise.allSettled`
- Return only successful results (skip failures silently)
- Used by company context agent for multi-page crawl

**Create `apps/mesh/src/diagnostic/agents/seo.ts`** — SEO diagnostic agent:

```typescript
async function runSeoAgent(crawl: CrawlResult): Promise<SeoResult>
```

Implementation — parse HTML using regex (no heavy DOM parser dependency, HTML is from fetch):
- **Title**: Extract `<title>` tag content
- **Meta description**: Extract `<meta name="description" content="...">`
- **OG tags**: Extract all `<meta property="og:*" content="...">` into a Record
- **Canonical URL**: Extract `<link rel="canonical" href="...">`
- **Heading structure**: Extract all `<h1>` through `<h6>` tags with text content (first 20 headings)
- **Robots meta**: Extract `<meta name="robots" content="...">`
- **Robots.txt check**: Fetch `{origin}/robots.txt` — set `hasRobotsTxt` to true if 200 response with non-empty body
- **Sitemap detection**: Check robots.txt for `Sitemap:` directive, then try `{origin}/sitemap.xml` — set `hasSitemap` accordingly
- **Structured data**: Extract all `<script type="application/ld+json">` blocks, parse each as JSON, return as array (skip parse failures)

Return a complete `SeoResult` object with all fields. Set fields to undefined if not found (not null — undefined means "checked but not present").

**Create `apps/mesh/src/diagnostic/agents/web-performance.ts`** — Web Performance diagnostic agent:

```typescript
async function runWebPerformanceAgent(url: string, crawl: CrawlResult): Promise<WebPerformanceResult>
```

Implementation:

**PSI API calls** — Two parallel calls (mobile + desktop):
```
GET https://www.googleapis.com/pagespeedonline/v5/runPagespeed?url={url}&strategy={mobile|desktop}&category=performance&key={optional}
```
- Read `PAGESPEED_API_KEY` from `process.env` (optional — works without but rate-limited)
- Timeout: 60 seconds per call (PSI can be slow, generous per user decision)
- Parse response to extract:
  - `lighthouseResult.categories.performance.score` (0-1, multiply by 100)
  - `lighthouseResult.audits['largest-contentful-paint'].numericValue` → LCP in ms
  - `lighthouseResult.audits['cumulative-layout-shift'].numericValue` → CLS
  - `lighthouseResult.audits['interaction-to-next-paint'].numericValue` → INP (if available)
  - `loadingExperience.metrics` → CrUX field data (LARGEST_CONTENTFUL_PAINT_MS, etc.)
- Rating thresholds (from web.dev):
  - LCP: good < 2500ms, poor > 4000ms
  - INP: good < 200ms, poor > 500ms
  - CLS: good < 0.1, poor > 0.25

**CrUX data extraction**:
- PSI API response includes `loadingExperience.metrics` which IS CrUX field data
- If `loadingExperience.overall_category` exists → site has CrUX data
- If not → fall back to `lighthouseResult.audits` lab data only
- Store CrUX data as-is in `cruxData` field
- This satisfies DIAG-04 (CrUX real-user data with PSI field data fallback) without a separate CrUX API call — PSI already embeds it

**Image audit** (from crawl HTML):
- Count total `<img>` tags
- Check for `loading="lazy"` attribute usage
- Check for `fetchpriority="high"` on above-fold images
- Check for `<link rel="preload" as="image">` tags — flag if more than 3 (too many preloads)
- Check image formats: detect AVIF, WebP usage from `src` and `srcset` attributes
- Check for `srcset` and `sizes` attributes (responsive images)
- Return issues array with severity levels

**HTML size analysis** (from crawl HTML):
- `totalBytes`: byte length of the full HTML
- `frameworkPayloadBytes`: size of `<script id="__NEXT_DATA__">`, `<script id="__FRSH_STATE">`, or similar framework JSON blobs
- `structuredDataBytes`: total size of all `<script type="application/ld+json">` blocks

**Cache header analysis** (from crawl headers):
- Extract `cache-control`, `etag`, `last-modified`, `x-cache`, `age`, `expires` headers
- Return as Record<string, string>

If PSI API call fails for a strategy, set that strategy's score to undefined (don't fail the whole agent). Return partial results.

**Create `apps/mesh/src/diagnostic/agents/company-context.ts`** — Company Context agent:

```typescript
async function runCompanyContextAgent(url: string, crawl: CrawlResult): Promise<CompanyContextResult>
```

Implementation:

**Multi-page crawl**:
- Start with the homepage HTML (from `crawl` parameter — already fetched)
- Parse navigation links from HTML: look for `<nav>` elements, extract `<a href="...">` links
- Filter to same-origin links only
- Prioritize pages with keywords: "about", "company", "who-we-are", "sobre", "quem-somos", "nossa-historia" (support Portuguese since many VTEX stores are Brazilian)
- Use `crawlMultiplePages` from `crawl.ts` to fetch up to 3 additional pages
- Timeout: 15 seconds per additional page

**LLM call for description generation**:
- Import `generateText` from `"ai"`
- Create LLM provider based on `DIAGNOSTIC_LLM_PROVIDER` env var:
  - If `"openai"` → dynamically import `@ai-sdk/openai` and create `openai(model)`
  - If `"anthropic"` → dynamically import `@ai-sdk/anthropic` and create `anthropic(model)`
  - If `"google"` → dynamically import `@ai-sdk/google` and create `google(model)`
  - Default to `"openai"` with model `"gpt-4o-mini"` if env vars not set
  - Read `DIAGNOSTIC_LLM_API_KEY` for the API key, `DIAGNOSTIC_LLM_MODEL` for model ID
- **IMPORTANT**: If no LLM API key is configured, skip the LLM call entirely and return null for the description. Log a warning: `"[diagnostic:company-context] No LLM API key configured, skipping AI company description"`
- System prompt for LLM:

```
You are analyzing an e-commerce storefront. Based on the following homepage and additional page content, write a company context description.

Write exactly two paragraphs in a conversational, friendly tone — like explaining this business to a colleague:
- First paragraph: What does this store sell? Who is their target customer?
- Second paragraph: What's their competitive positioning? What market category are they in?

Keep it concise — 3-4 sentences per paragraph maximum. Be specific about products and audience. If you can't determine something, say so briefly rather than guessing.
```

- User prompt: concatenate homepage content (first 3000 chars of body text, stripped of HTML tags) plus any additional page content (first 1500 chars each)
- `maxOutputTokens`: 500
- `temperature`: 0.3

**Extract additional signals**:
- `productSignals`: look for product-related schema.org types, product grids, price patterns
- `targetAudience`: infer from content, categories, product types
- `competitiveAngle`: from "about" page content or brand messaging
- `crawledPages`: list of URLs that were successfully crawled

If LLM call fails, return result with description as undefined but include other extracted signals.
