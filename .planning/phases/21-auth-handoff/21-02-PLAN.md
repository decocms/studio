```typescript
function SignupCTA({ token }: { token: string }) {
  const session = authClient.useSession();

  // Store token in sessionStorage as fallback for OAuth redirects
  // that may strip query params during multi-step redirect chains.
  // This runs on every render — sessionStorage is synchronous and fast.
  if (typeof window !== "undefined") {
    sessionStorage.setItem(
      LOCALSTORAGE_KEYS.onboardingToken(),
      token,
    );
  }

  // Don't show CTA if user is already logged in
  if (session.data) {
    return null;
  }

  const loginUrl = `/login?next=${encodeURIComponent(`/onboard-setup?token=${token}`)}`;

  return (
    <div className="rounded-xl border-2 border-brand/30 bg-brand/5 p-6 text-center">
      <h3 className="text-lg font-semibold text-foreground">
        Save this report to your team
      </h3>
      <p className="mt-1 text-sm text-muted-foreground">
        Sign up to keep this diagnostic, track improvements over time, and unlock AI-powered recommendations for your store.
      </p>
      <a
        href={loginUrl}
        className="mt-4 inline-flex items-center rounded-lg bg-brand px-5 py-2.5 text-sm font-semibold text-brand-foreground transition-opacity hover:opacity-90"
      >
        Sign up free
      </a>
    </div>
  );
}
```

**Key design decisions:**
- Use `authClient.useSession()` (already imported from the auth-client module) to check if user is logged in — if logged in, hide the CTA. This is a React hook call, not useEffect.
- The `?next=` param points to `/onboard-setup?token=<token>` — this is the post-login setup page (created in Plan 03). After login, Better Auth redirects to `next`, which will be the onboard-setup page with the token.
- `sessionStorage` stores the token as a fallback. Some OAuth providers (e.g., Google) may strip custom params during multi-step redirects. The onboard-setup page will check sessionStorage if the URL param is missing.
- The CTA does NOT use useEffect — the sessionStorage write is synchronous and runs during render (React 19 compatible). It's idempotent (same token, same key).
- Use `encodeURIComponent` for the `next` value since it contains `?` and `=` characters.
