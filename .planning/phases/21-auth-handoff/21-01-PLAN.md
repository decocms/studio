  <action>
Create `apps/mesh/src/api/routes/onboarding.ts` following the factory pattern from `createDiagnosticRoutes`:

```typescript
export function createOnboardingRoutes(db: Kysely<Database>, authInstance: typeof auth)
```

Takes `db` and `auth` instance as parameters (no MeshContext). Creates a Hono app with two routes:

**Helper: `getSessionOrFail(c, authInstance)`**
- Extract session from request using `authInstance.api.getSession({ headers: c.req.raw.headers })`
- If no session, return `c.json({ error: "Authentication required" }, 401)`
- Return the session user (id, email, name)

**GET /resolve** (query param: `token`)
1. Check auth session via helper
2. Load diagnostic session by token via `DiagnosticSessionStorage.findByToken(token)`
3. If not found → 404
4. Extract company name from `session.results.companyContext?.description` — parse the first sentence or use the first line as the org name candidate. Specifically: take the AI description text, extract the company/brand name from it. A simple heuristic: the description typically starts with "CompanyName is..." — extract the word(s) before " is " or " are ". Fallback: use the hostname from `session.url` (e.g., "example.com" → "example").
5. Extract the user's email domain (user.email.split("@")[1])
6. Query existing organizations where the slug or name contains the email domain — use `authInstance.api.listOrganizations({ headers })` to get orgs the user can see, plus query the database directly for orgs matching the email domain:
   ```sql
   SELECT id, name, slug FROM organization WHERE slug LIKE '%domain%' OR name LIKE '%domain%' LIMIT 5
   ```
7. Return JSON:
   ```json
   {
     "session": { "token": "...", "url": "...", "companyName": "..." },
     "suggestedOrgName": "extracted company name",
     "matchingOrgs": [{ "id": "...", "name": "...", "slug": "...", "memberCount": 3 }]
   }
   ```

**POST /claim** (body: `{ token, action: "create" | "join", orgId?: string }`)
1. Check auth session via helper
2. Load diagnostic session by token — 404 if not found
3. If session already has `organizationId` → 409 "Session already claimed"
4. **If action === "create":**
   a. Get the company name (same extraction logic as resolve, or accept it in the body as `orgName`)
   b. Slugify it (import slugify from auth/index.ts — IMPORTANT: export the slugify function from auth/index.ts so it can be reused, or duplicate the simple function locally)
   c. Create org via `authInstance.api.createOrganization({ body: { name, slug, userId } })`
   d. On conflict (slug exists), append random suffix and retry (same pattern as auth/index.ts databaseHooks)
   e. After org creation, `seedOrgDb` is called automatically by the Better Auth afterCreate hook
   f. Set the new org as active via `authInstance.api.setActiveOrganization({ headers, body: { organizationId } })`
5. **If action === "join":**
   a. Require `orgId` in body — 400 if missing
   b. Verify the org exists (not strictly needed if Better Auth handles it)
   c. Set as active org
   d. Note: the user should already be a member (they'd need to be invited or self-join). For now, just set active and proceed. If they're not a member, Better Auth will error — return that error.
6. **Create project** for the storefront URL:
   a. Use `ProjectsStorage.create({ organizationId, slug, name, description })`:
      - slug: derive from the storefront URL hostname (e.g., "mystore-com")
      - name: storefront URL hostname (e.g., "mystore.com")
      - description: "Storefront diagnostic for {url}"
   b. If project creation fails (slug conflict), append random suffix and retry
7. **Associate session**: Call `storage.associateOrg(token, organizationId, projectId)`
8. Return JSON:
   ```json
   {
     "organizationId": "...",
     "organizationSlug": "...",
     "projectId": "...",
     "projectSlug": "..."
   }
   ```

**Important implementation notes:**
- Use `import { auth } from "../../auth"` for the auth instance type, but receive it as parameter (same pattern as diagnostic routes)
- Import `DiagnosticSessionStorage` from storage
- Import `ProjectsStorage` from storage
- Import `Database` type from storage/types
- The org creation uses Better Auth's `auth.api.createOrganization` which triggers the `afterCreate` hook (seedOrgDb) automatically
- Duplicate the `slugify` function locally in this file (it's a simple 5-line function, avoids exporting from auth/index.ts which has complex initialization)
- Add a simple `urlToSlug(url: string)` helper: parse URL, take hostname, replace dots with hyphens, lowercase
