**1. Create the interview page (`apps/mesh/src/web/routes/onboard-interview.tsx`)**

This page renders AFTER the user completes org setup (redirected from /onboard-setup).

The interview uses `useChat` from `@ai-sdk/react` directly (NOT the ChatProvider from chat/context.tsx which requires Virtual MCP selection, model selection, thread management etc. — the interview is a simpler, self-contained chat).

Key implementation:
- Route receives `?org=<orgSlug>&token=<diagnosticToken>` search params (passed from onboard-setup claim success)
- Auth guard: use `authClient.useSession()` — if not authenticated, redirect to `/login` via `<Navigate>`
- Load diagnostic results via `fetch(/api/diagnostic/session/${token})` to display context about the storefront
- Use `useChat` from `@ai-sdk/react` with:
  - `api`: `/api/${org}/decopilot/stream` (existing endpoint)
  - Send the first message automatically on mount as a user message that includes the diagnostic context, so the LLM can start asking relevant questions. The user message text should be something like: "I just ran a diagnostic on my storefront [URL]. Here's what we found: [summary of diagnostic results]. Help me set up the right agents for my store." This gives the LLM context to ask targeted questions.
  - Include in message metadata: `system` field with a structured onboarding interview system prompt
  - The system prompt instructs the LLM to:
    - Act as an onboarding consultant for e-commerce stores
    - Ask exactly 3 focused questions, one at a time, about: (1) their top business goal, (2) their biggest challenge, (3) what they'd automate first
    - Keep each question concise (1-2 sentences)
    - After the 3rd answer, summarize the findings in a structured format with a special marker: `[INTERVIEW_COMPLETE]` followed by JSON: `{"goals": [...], "challenges": [...], "priorities": [...]}`
    - Be conversational, not robotic — reference their diagnostic data
  - The metadata must include `agent` (use the default decopilot Virtual MCP ID), `models` (need to resolve from available models), and `thread_id`

- For model resolution: Since this is an onboarding page and we don't want to burden the user with model selection, fetch allowed models via `GET /api/${org}/decopilot/allowed-models` and pick the first available. Store in a ref. If no models available, show a helpful error.

- Track message count: Count user messages. After the user has sent 3 messages (their answers to the 3 questions), watch for the assistant's response containing `[INTERVIEW_COMPLETE]`. Parse the JSON payload from the response.

- On interview completion:
  - Extract the structured JSON from the assistant's final message
  - Call `POST /api/onboarding/interview-results` with `{ token, goals, challenges, priorities }`
  - Navigate to `/onboard-interview?org=${org}&token=${token}&step=recommendations` (same page, different view state — plan 03 will render the recommendations view)

- UI layout:
  - Full-screen centered layout (same pattern as onboard-setup.tsx)
  - Header: "Tell us about your store" with the storefront URL below
  - Chat messages area: Simple message list (assistant messages left-aligned with subtle background, user messages right-aligned)
  - Input area: Text input + send button at bottom. Disable input while streaming.
  - Progress indicator: Show "Question N of 3" above the input
  - After completion: Show a brief "Analyzing your responses..." loading state before transitioning

- Important React 19 rules: NO useEffect, NO useMemo/useCallback/memo. Use `useChat` from `@ai-sdk/react` directly (this is a library hook, not the banned project useEffect). The automatic first message can be triggered using `initialMessages` prop of useChat or by calling `sendMessage` during a user interaction handler.

**2. Add interview-results API endpoint (`apps/mesh/src/api/routes/onboarding.ts`)**

Add a new POST route inside the existing `createOnboardingRoutes` factory:

`POST /interview-results` — Persists interview results to the org's company context:
- Auth check (same pattern as /resolve and /claim)
- Validate body: `{ token: string, organizationId: string, goals: string[], challenges: string[], priorities: string[] }`
- Load the diagnostic session by token
- Verify the session belongs to the authenticated user's org
- Update the diagnostic session results by adding an `interviewResults` field to the results JSON (use `sessionStorage.updateResults()` if available, or direct DB update)
- Also: store a structured company context document that combines the AI-generated description with the interview findings. This should be queryable later by the recommendation engine.
- Return 200 with `{ success: true }`
