<task type="auto">
  <name>Task 2: Build recommendation engine + API endpoint</name>
  <files>apps/mesh/src/api/routes/onboarding.ts</files>
  <action>
Add a `GET /recommendations?token=...&organizationId=...` endpoint inside the existing `createOnboardingRoutes` factory function.

**Endpoint logic:**

1. **Auth check** — same `getSessionUser(c)` pattern as /resolve and /claim
2. **Validate query params** — `token: string, organizationId: string` (both required)
3. **Load diagnostic session** by token — get `results` (diagnostic data + interview results)
4. **Load Virtual MCPs** for the organization — query the database directly using Kysely:
   ```typescript
   const virtualMcps = await db
     .selectFrom("virtual_mcp")
     .selectAll()
     .where("organization_id", "=", organizationId)
     .where("status", "=", "active")
     .execute();
   ```
   Then load their connections from `virtual_mcp_connection` table to build full VirtualMCPEntity-like objects. Check the existing virtual MCP storage patterns in `apps/mesh/src/storage/virtual-mcps.ts` for the correct table names and join pattern.

5. **Score each Virtual MCP** against the diagnostic data + interview results using rule-based matching:

   The scoring function `scoreAgent(virtualMcp, diagnosticResults, interviewResults)` returns `{ score: number, reasons: string[] }`:

   **Keyword matching (against Virtual MCP title + description + metadata.instructions):**
   - Performance/speed/LCP/CWV keywords → match if `webPerformance` shows poor scores (LCP poor/needs-improvement, or mobile/desktop score < 70)
   - SEO/meta/search/ranking keywords → match if `seo` results show missing meta description, missing OG tags, no sitemap, no robots.txt
   - Content/copy/description keywords → match if `companyContext` description is short or missing
   - Platform-specific keywords (VTEX, Shopify, etc.) → match if `techStack.platform.name` matches
   - E-commerce/store/product/catalog keywords → always a mild positive (these are storefront users)

   **Interview goal matching:**
   - For each goal/challenge/priority string from interview results, check if any word appears in the Virtual MCP's title/description/instructions
   - Each keyword match adds to score

   **Scoring weights:**
   - Diagnostic issue match (poor performance, missing SEO, etc.): +30 per match
   - Interview keyword match (goal/challenge mentions agent's domain): +20 per match
   - Platform match (agent mentions user's detected platform): +25
   - Base relevance (e-commerce keywords in agent): +10

   **Reason generation:**
   - For each match, build a plain-English reason: "Your site's LCP is rated 'poor' — this agent can help optimize loading performance"
   - Combine top 1-2 reasons into the recommendation's `reason` field

6. **Filter and sort:**
   - Exclude any Virtual MCP with score 0 (no relevance signal at all)
   - Exclude the default decopilot Virtual MCP (filter using `isDecopilot` pattern — check if the ID matches the well-known decopilot ID format)
   - Sort by score descending
   - Take top 3

7. **Build response:** For each recommended Virtual MCP, include:
   - `agentId`, `agentTitle`, `agentDescription`, `agentIcon` from the Virtual MCP entity
   - `reason` — the plain-English explanation (combine top reasons)
   - `score` — the numeric score
   - `requiredConnections` — map from the Virtual MCP's connections array. For each connection, look up the connection entity to get its title. Mark `isConfigured: true` if the connection exists and has a `connection_url` or `connection_headers` (i.e., it's actually wired up, not just a stub).

8. **Fallback:** If no agents score above 0 (e.g., new org with no Virtual MCPs), return an empty array. The UI (plan 03) will handle this gracefully.

9. **Return** `200` with `{ recommendations: AgentRecommendation[] }`

**Important:** This endpoint runs inside `createOnboardingRoutes` which is pre-MeshContext. Access the database directly via the `db` parameter (Kysely instance). Do NOT use `ctx.storage` (no MeshContext available). Look at the Virtual MCP storage module to find the correct table names and column names.
