  <action>
    Create `apps/mesh/src/web/routes/onboarding.tsx` — the public onboarding page.

    **Important constraints:**
    - React 19 — NO useEffect, NO useMemo/useCallback/memo (banned by lint plugins)
    - Tailwind v4 design system tokens
    - Use `@tanstack/react-query` for data fetching (useQuery with refetchInterval for polling)
    - Import diagnostic types from `@/diagnostic/types` (DiagnosticSession, AgentStatus, DiagnosticAgentId)
    - Import KEYS from `@/web/lib/query-keys`
    - Use `useNavigate` from `@tanstack/react-router` for programmatic navigation to `/report/$token`

    **Component structure:**

    The page has two states managed by React state:

    1. **URL Input State** (`token === null`):
       - Clean, minimal page with centered layout
       - Heading: "Storefront Diagnostic" (or similar)
       - Subheading: brief description like "Enter your store URL to get an instant performance report"
       - URL input field (`<input type="url">`) with placeholder "https://your-store.com"
       - Submit button ("Analyze" or "Run Diagnostic")
       - On submit: POST to `/api/diagnostic/scan` with `{ url }`, get back `{ token }`, set token state
       - Show inline error if POST returns error (validation, rate limit)
       - While submitting, disable button and show loading indicator

    2. **Loading State** (`token !== null`, session not complete):
       - Use `useQuery` with `KEYS.diagnosticSession(token)` as key
       - queryFn: `fetch(/api/diagnostic/session/${token}).then(r => r.json())`
       - `refetchInterval: 1500` (poll every 1.5s while session is running)
       - `refetchInterval` should be `false` when session.status is "completed" or "failed"
       - Display: "Analyzing {url}..." heading
       - Agent checklist showing 4 agents with their status:
         - Map over the session's `agents` object (keys are DiagnosticAgentId)
         - Display names: "Web Performance", "SEO Analysis", "Tech Stack Detection", "Company Context"
         - Status indicators: spinner for "pending"/"running", checkmark for "completed", X for "failed"
         - Use a simple mapping object for display names: `{ web_performance: "Web Performance", seo: "SEO Analysis", tech_stack: "Tech Stack Detection", company_context: "Company Context" }`
       - When session.status === "completed" or "failed": call `navigate({ to: "/report/$token", params: { token } })`
       - Since useEffect is banned: use a pattern where navigation happens inline in the render. If the query data shows completed/failed, render a component that triggers navigation. One approach: check the condition and call `navigate()` directly in the component body during render — TanStack Router's navigate is safe to call during render as it's a redirect. Alternatively, use `<Navigate to="/report/$token" params={{ token }} />` from TanStack Router.

    **Styling guidelines:**
    - Full-page layout: `min-h-screen` with a clean background
    - Use the same brand gradient feel as the login page for consistency (look at login.tsx styling)
    - Or a neutral white/light gray for a "document/report" feel — Claude's discretion
    - Input should be prominent — large, clean, centered
    - Agent checklist should have clear visual hierarchy — agent names left, status icons right
    - Use `text-foreground`, `bg-background`, `border-border` tokens, etc. from Tailwind v4 design system

    **Error handling:**
    - Show validation errors inline below the input
    - If scan POST fails with 429, show "Too many requests" message
    - If session polling shows "failed" status, still navigate to report page (it will show partial results)
